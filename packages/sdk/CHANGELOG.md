## 0.2.0

### Minor Changes

- SigNoz/OpenTelemetry correctness fixes and OpenTelemetry 2.x upgrade:

  - **Metrics now export with delta temporality** (SigNoz's recommendation)
    instead of OpenTelemetry's default cumulative. `detectBehavioralDrift` sums
    `agent.tool.calls` / `agent.tokens` and takes p95 of `agent.tool.duration`
    over time windows — summing cumulative running totals produced meaningless,
    ever-growing numbers; delta makes the windowed queries correct.
  - **Added `TelemetryConfig.otlpHeaders`** (env `OTEL_EXPORTER_OTLP_HEADERS`,
    standard `k=v,k2=v2` form), attached to every OTLP export. Enables **SigNoz
    Cloud** via `signoz-ingestion-key=<key>`; self-hosted needs none.
  - **`bootstrapTelemetry` now exports logs** over OTLP (`/v1/logs`) alongside
    traces and metrics, and enables pino log↔trace correlation (`trace_id` /
    `span_id` on Fastify logs), so you can jump from a span to its logs.
  - Upgraded the OpenTelemetry stack to 2.x (`resources` / `sdk-metrics` 2.9,
    `sdk-node` / exporters / `sdk-logs` 0.220, `auto-instrumentations-node`
    0.78) and switched from the deprecated `new Resource()` to
    `resourceFromAttributes()`.

  Also folds in prior unreleased changes shipping in this release: the package
  rename from `@agentpulse/sdk` to `@driftwatch/sdk` (config API
  `AgentPulseConfigSchema` / `loadAgentPulseConfigFromEnv` →
  `DriftWatchConfigSchema` / `loadDriftWatchConfigFromEnv`),
  `TelemetryConfig.capturePayloads` (env `OTEL_CAPTURE_PAYLOADS`), and
  `bootstrapTelemetry`'s `{ registerShutdownHandlers: false }` option.
